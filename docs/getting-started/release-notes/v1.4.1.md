# Notes de release — v1.4.1

**Date** : juin 2026  
**Image** : `ghcr.io/arcneell/caelix:1.4.1` (également `:latest`)

---

## Résumé

La v1.4.1 **durcit et assouplit l'installation** pour qu'elle s'adapte à toutes les situations. Aucun changement fonctionnel du moteur — c'est une release de correctifs d'installation.

## Nouvelles options d'installation

L'installeur (`install.sh`) accepte désormais :

| Option | Défaut | Rôle |
|--------|--------|------|
| `--ui-port PORT` | `18100` | Port d'écoute de la console web |
| `--ui-bind ADDR` | `0.0.0.0` | Adresse d'écoute (`127.0.0.1` = local uniquement) |
| `--lang fr\|en` | `fr` | Langue CLI / notifications |

Chacune a aussi un équivalent en variable d'environnement (`CAELIX_UI_PORT`, `CAELIX_UI_BIND`, `CAELIX_LANG`).

```bash
docker run --rm ghcr.io/arcneell/caelix:latest cat /opt/caelix/install.sh \
  | bash -s -- --with-systemd --ui-port 9000 --ui-bind 127.0.0.1
```

## Corrections de robustesse

- **Manifeste généré correctement** : le port, l'adresse, l'image (= celle réellement tirée) et la racine sont substitués dans `manifest.ini` (avant : port et image figés). Ajout de `CAELIX_HOST_ROOT`.
- **Unité systemd adaptative** :
  - `AmbientCapabilities=CAP_NET_BIND_SERVICE` → le proxy peut écouter sur `:80`/`:443` (avant : `NoNewPrivileges` bloquait le bind privilégié) ;
  - `ReadWritePaths` → l'état `.caelix/` reste inscriptible ;
  - `ProtectHome` désactivé automatiquement quand l'installation est sous `/home` (évitait l'échec d'écriture).
- **Accès Docker** : si l'utilisateur n'est pas dans le groupe `docker`, bascule automatique sur `sudo docker` pour l'installation (au lieu d'échouer en cours de route).
- **Validations** : port numérique (1-65535), langue, chemin d'install absolu ; **avertissement si le port choisi est déjà occupé**.
- Messages et bannières corrigés (plus de codes couleur en clair).
