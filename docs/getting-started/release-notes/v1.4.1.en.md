# Release Notes — v1.4.1

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/caelix:1.4.1` (also `:latest`)

---

## Summary

v1.4.1 **hardens and loosens the installation** so it adapts to any situation. No engine behaviour change — this is an install-fix release.

## New install options

The installer (`install.sh`) now accepts:

| Option | Default | Purpose |
|--------|---------|---------|
| `--ui-port PORT` | `18100` | Web console listen port |
| `--ui-bind ADDR` | `0.0.0.0` | Bind address (`127.0.0.1` = local only) |
| `--lang fr\|en` | `fr` | CLI / notifications language |

Each also has an environment-variable equivalent (`CAELIX_UI_PORT`, `CAELIX_UI_BIND`, `CAELIX_LANG`).

```bash
docker run --rm ghcr.io/arcneell/caelix:latest cat /opt/caelix/install.sh \
  | bash -s -- --with-systemd --ui-port 9000 --ui-bind 127.0.0.1
```

## Robustness fixes

- **Manifest generated correctly**: port, bind address, image (= the one actually pulled) and root are substituted into `manifest.ini` (previously the port and image were hard-coded). Added `CAELIX_HOST_ROOT`.
- **Adaptive systemd unit**:
  - `AmbientCapabilities=CAP_NET_BIND_SERVICE` → the proxy can listen on `:80`/`:443` (previously `NoNewPrivileges` blocked privileged binds);
  - `ReadWritePaths` → the `.caelix/` state stays writable;
  - `ProtectHome` automatically disabled when installing under `/home` (which used to break state writes).
- **Docker access**: if the user is not in the `docker` group, automatically falls back to `sudo docker` for the install (instead of failing mid-run).
- **Validations**: numeric port (1-65535), language, absolute install path; **warns if the chosen port is already in use**.
- Fixed messages and banners (no more raw colour codes).

## Upgrade

The `:latest` tag points to this version — the usual install command pulls the fix directly.
