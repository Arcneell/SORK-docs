# Release Notes — v1.3.0

**Date**: June 2026  
**Image**: `ghcr.io/arcneell/sork:1.3.0` (also `:latest`)

---

## Summary

v1.3.0 consolidates SORK after several critical fix cycles across the engine, web console, and distribution pipeline. This release adds the guided deployment assistant (27 templates), backend security hardening, autoscale/blue-green fixes, GitHub Actions CI, and a complete Docker distribution pipeline.

!!! tip "Updating from a previous version"
    The files `etc/manifest.ini` and `etc/notify.ini` are **never overwritten** by the installer. After updating the image, run `bin/sork once` or restart the systemd service. Increment `config_version` for `[sork-ui]` if you rebuild the UI locally.

---

## New Features

### Assistant & App Store
- Guided assistant with **27 stack templates** (Portainer-compatible)
- Smart deployment: env vars, naming, health checks, autoscale, volumes
- Hardened deploy wizard (integrity, progress, error handling)

### Domains & Proxy
- Automatic proxy port assignment when adding a domain
- Global proxy auto-start on domain apply
- Proxy relay fix, random ports, `autoscale_route` routing
- Container ports bound on `0.0.0.0` (headless servers)

### Distribution
- Single Docker image `ghcr.io/arcneell/sork` (bytecode backend, minified frontend, Bash engine)
- Embedded installer in the image (`install.sh`) — no GitHub access required on client side
- UI bound to `0.0.0.0` by default, server IP shown at install time
- Cleanup of existing SORK containers before first reconciliation
- systemd / `sork once` race condition fix
- Sourceless Python bytecode (`compileall -b`)

### CI & Quality
- GitHub Actions: shellcheck, `check-all`, frontend build, backend validation
- Portable `check-i18n.sh` (no `grep -P`)
- Bash ≥ 4 guard in `check-all.sh`

## Critical Fixes (Engine)

| Area | Fix |
|------|-----|
| **Autoscale** | Glob replica matching (app names with dots), port release on scale-down, cleanup when `autoscale=0`, auto-update rollout on replicas |
| **Blue/green** | `*-candidate-*` containers protected from orphan removal |
| **Global proxy** | Auto-stop when `[proxy]` section removed from manifest |
| **Health checks** | URL rewrite for probes from SORK container (`_sork_rewrite_url`) |
| **Concurrency** | Atomic heartbeat, `fail_count` counters protected by `flock` |
| **Applications** | Health checks disabled on DB/cache, backup spam fix on databases |

## Backend & UI Fixes

- JWT auth hardening, deploy integrity, manifest env keys
- Reconciliation triggerable from UI
- Bilingual FR/EN notifications, incidents → UI bridge
- Loading spinner on delete confirmation modal
- Repaired stack templates (previously failing deploys)

## Unchanged Requirements

- Bash 5.x, Docker or Podman
- `socat` on host for autoscale/proxy
- Python 3.11+ (in distribution image)

## Install / Upgrade

```bash
# Registry authentication
echo "TOKEN" | docker login ghcr.io -u Arcneell --password-stdin

# Install or upgrade
docker run --rm ghcr.io/arcneell/sork:1.3.0 cat /opt/sork/install.sh | bash -s -- --with-systemd
```

See [Distribution & Release](../distribution.en.md) and [Installation](../installation.en.md).
