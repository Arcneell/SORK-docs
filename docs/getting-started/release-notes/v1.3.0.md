# Notes de release — v1.3.0

**Date** : juin 2026  
**Image** : `ghcr.io/arcneell/sork:1.3.0` (également `:latest`)

---

## Résumé

La v1.3.0 consolide SORK après plusieurs cycles de correctifs critiques sur le moteur, la console web et la distribution. Cette release apporte l'assistant de déploiement (27 templates), le durcissement sécurité backend, des correctifs autoscale/blue-green, la CI GitHub Actions, et une pipeline de distribution Docker complète.

!!! tip "Mise à jour depuis une version antérieure"
    Les fichiers `etc/manifest.ini` et `etc/notify.ini` ne sont **jamais écrasés** par l'installeur. Après mise à jour de l'image, relancez `bin/sork once` ou redémarrez le service systemd. Incrémentez `config_version` pour `[sork-ui]` si vous rebuild l'UI localement.

---

## Nouveautés

### Assistant & App Store
- Assistant guidé avec **27 templates** de stacks (Portainer-compatible)
- Déploiement intelligent : variables d'environnement, nommage, health checks, autoscale, volumes
- Wizard de déploiement durci (intégrité, progression, erreurs)

### Domaines & proxy
- Assignation automatique du port proxy lors de l'ajout d'un domaine
- Démarrage automatique du proxy global à l'application d'un domaine
- Correction du relay proxy, ports aléatoires, routage `autoscale_route`
- Bind des ports conteneur sur `0.0.0.0` (serveurs headless)

### Distribution
- Image Docker unique `ghcr.io/arcneell/sork` (backend bytecode, frontend minifié, moteur Bash)
- Installeur embarqué dans l'image (`install.sh`) — pas d'accès GitHub requis côté client
- Bind UI `0.0.0.0` par défaut, IP serveur affichée à l'installation
- Nettoyage des conteneurs SORK existants avant première réconciliation
- Correction race condition systemd / `sork once`
- Bytecode Python sourceless (`compileall -b`)

### CI & qualité
- GitHub Actions : shellcheck, `check-all`, build frontend, validation backend
- Script `check-i18n.sh` portable (sans `grep -P`)
- Garde Bash ≥ 4 dans `check-all.sh`

## Correctifs critiques (moteur)

| Domaine | Correction |
|---------|------------|
| **Autoscale** | Matching réplicas en glob (noms d'app avec points), libération ports au scale-down, cleanup si `autoscale=0`, rollout auto-update sur réplicas |
| **Blue/green** | Candidats `*-candidate-*` protégés contre suppression orphelins |
| **Proxy global** | Arrêt automatique si section `[proxy]` retirée du manifeste |
| **Health checks** | Réécriture URL pour probes depuis conteneur SORK (`_sork_rewrite_url`) |
| **Concurrence** | Heartbeat atomique, compteurs `fail_count` protégés par `flock` |
| **Applications** | Health checks désactivés sur BDD/cache, fin du spam backup sur bases de données |

---

## Correctifs backend & UI

- Durcissement auth JWT, intégrité déploiement, clés env manifeste
- Réconciliation déclenchable depuis l'UI
- Notifications bilingues FR/EN, bridge incidents → UI
- Spinner de chargement sur modal de suppression
- Templates stacks réparés (déploiements qui échouaient)

## Prérequis inchangés

- Bash 5.x, Docker ou Podman
- `socat` sur l'hôte pour autoscale/proxy
- Python 3.11+ (dans l'image de distribution)
